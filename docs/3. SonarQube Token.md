
```groovy
stage('SAST') {
  steps {
    withSonarQubeEnv('sonarqube') {
      sh 'sonar-scanner'
    }
  }
}
```

For this stage to work, **4 things must be configured**:

1️⃣ SonarQube server running
2️⃣ SonarQube token generated
3️⃣ Token added in Jenkins credentials
4️⃣ SonarQube tool configured in Jenkins

I will explain **every step**.

---

# PART 1 — Start SonarQube Server

Login to your EC2 server.

Run SonarQube container:

```bash
docker run -d \
--name sonarqube \
-p 9000:9000 \
sonarqube:lts-community
```

Check container:

```bash
docker ps
```

You should see something like:

```
sonarqube
```

---

# PART 2 — Open SonarQube Dashboard

Open browser:

```
http://EC2_PUBLIC_IP:9000
```

Default login:

```
username: admin
password: admin
```

After login SonarQube will ask you to **change the password**.

Example new password:

```
Admin@123
```

---

# PART 3 — Generate Sonar Token

After login:

1️⃣ Look at **top right corner**

Click your **profile icon**.

Then click:

```
My Account
```

---

Now open the tab:

```
Security
```

You will see **Generate Tokens**.

---

# PART 4 — Create Token

Enter token name:

```
jenkins-token
```

Click:

```
Generate
```

SonarQube will show a token like:

```
sqp_6c3d4fxxxxxxxxxxxxxxxxxxxx
```

⚠️ Copy this token immediately
You **cannot see it again later**.

This is your **sonar-token**.

---

# PART 5 — Add Token to Jenkins

Open Jenkins:

```
http://EC2_PUBLIC_IP:8080
```

Go to:

```
Manage Jenkins
→ Credentials
→ Global
→ Add Credentials
```

Fill like this:

| Field       | Value             |
| ----------- | ----------------- |
| Kind        | Secret Text       |
| Secret      | paste Sonar token |
| ID          | sonar-token       |
| Description | SonarQube Token   |

Click:

```
Create
```

Now Jenkins stores your token securely.

---

# PART 6 — Configure SonarQube Server in Jenkins

Now we connect Jenkins to SonarQube.

Go to:

```
Manage Jenkins
→ Configure System
```

Scroll until you find:

```
SonarQube Servers
```

Click:

```
Add SonarQube
```

Fill the fields:

| Field                       | Value                     |
| --------------------------- | ------------------------- |
| Name                        | sonarqube                 |
| Server URL                  | http://EC2_PUBLIC_IP:9000 |
| Server authentication token | sonar-token               |

Click:

```
Save
```

Important:

The name **must match** your pipeline:

```
withSonarQubeEnv('sonarqube')
```

---

# PART 7 — Install Sonar Scanner Tool

Now configure the scanner tool.

Go to:

```
Manage Jenkins
→ Global Tool Configuration
```

Scroll until you see:

```
SonarQube Scanner
```

Click:

```
Add SonarQube Scanner
```

Fill:

```
Name: sonar-scanner
```

Check:

```
Install automatically
```

Save.

# PART 9 — sonar-project.properties File

Your project must contain:

```
sonar-project.properties
```

Example:

```
sonar.projectKey=weather-app
sonar.projectName=weather-app
sonar.projectVersion=1.0
sonar.sourceEncoding=UTF-8
sonar.sources=src
sonar.tests=src
sonar.test.inclusions=src/**/*.test.js,src/**/*.test.jsx
sonar.exclusions=build/**,node_modules/**,public/**,coverage/**
sonar.javascript.lcov.reportPaths=coverage/lcov.info
```

This tells SonarQube **what to scan**.

---

# PART 10 — Add Sonar Stage in Pipeline

Your Jenkinsfile stage:

```groovy
stage('SAST') {
  steps {
    withSonarQubeEnv('sonarqube') {
      sh 'sonar-scanner'
    }
  }
}
```

Pipeline process:

```
Jenkins
   ↓
runs sonar-scanner
   ↓
scanner reads sonar-project.properties
   ↓
code sent to SonarQube
   ↓
SonarQube analyzes project
```

---

# PART 11 — Run Pipeline

Go to Jenkins pipeline.

Click:

```
Build Now
```

Pipeline will run.

When it reaches **SAST stage**, SonarQube analysis begins.
